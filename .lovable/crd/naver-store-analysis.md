# 네이버 스마트스토어/브랜드스토어 전용 분석 (보류 — AutoBlog 론칭 후 진행)

> **상태**: 📦 보관 / **재개 시점**: AutoBlog 정식 론칭 완료 후
> **포지션**: 마이크로 추가 기능 (메인 분석 흐름과 분리)

---

## 1. 배경 / 문제

- 고객사의 약 50%가 네이버 스마트스토어(`smartstore.naver.com`) 또는 브랜드스토어(`brand.naver.com`) URL을 사용함
- 네이버 도메인은 다음 이유로 일반 SEO/AEO/GEO 분석이 부정확:
  - 고정 템플릿 (자체 메타/구조 변경 불가)
  - 제한적인 robots.txt 정책
  - 결과적으로 점수가 0~30점대로 나오며, 사용자가 "도구가 이상하다"고 오해할 위험
- **방어 셋팅 또는 전용 분석 방식 도입 필요**

### 예시 URL
- https://brand.naver.com/mujikorea
- https://smartstore.naver.com/modern_m

## 4. 대체 콘텐츠 (3가지 조합)

### 4-1. GEO 점수 (정상 측정)
- 추출한 브랜드명 기반으로 AI 인용도 측정 (기존 GEO 로직 재사용)

### 4-2. 외부 언급 / 백링크
- Firecrawl search로 브랜드명 검색
- `naver.com` 도메인 제외, 외부 도메인 노출 카운트
- "○○ 사이트에서 N번 언급되고 있어요" 형태로 표시

### 4-3. AutoBlog CTA (메인 푸시)
- 위치: **기존 funnel CTA 영역**
- 메시지: "네이버 스토어는 한계가 있어요. 자체 사이트 + AutoBlog로 검색 트래픽 직접 확보하기"

### 4-4. SEO/AEO
- "네이버 정책상 진단 불가" 회색 배지 처리

---

## 5. 브랜드명 추출 파이프라인 (3단계 폴백)

1. **Firecrawl 스크랩** → `<title>`, `og:title`, `og:site_name` 파싱 (1차)
2. **Lovable AI 추출** → 1차 실패 시 markdown 던져서 Gemini Flash로 브랜드명 추출 (2차)
3. **URL 슬러그** → AI도 실패 시 슬러그 사용 + "정확하지 않을 수 있어요" 경고 (3차, 최후)
